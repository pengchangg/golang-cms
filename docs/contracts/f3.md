# F3 冻结契约

- 冻结版本：F3
- 状态：已冻结并集成
- 适用范围：阶段三 S3 兼容对象存储素材、媒体字段、同步 CSV 导入导出和素材客户端路由
- 依据：`docs/cms-design.md`、`docs/development-plan.md`、`docs/contracts/f0.md`、`docs/contracts/f1.md`、`docs/contracts/f2.md` 和当前阶段二实现
- OpenAPI 分片：`api/openapi/fragments/admin/assets/**`、`admin/transfers/**`、`content/assets/**`
- 集成状态：本文新增分片均标记 `x-integration-status: integrated` 并聚合进根 OpenAPI

F3 只做 F0 至 F2 的兼容扩展，不改变现有管理内容 DTO、客户端内容查询、分页、展开或 ETag 语义。本文冻结契约；实现可按本文聚合根 OpenAPI、注册运行时路由并修改 Go 代码、Web 代码或迁移。

## 1. S3 兼容对象存储边界和生产配置

### 1.1 私有 Bucket

- 素材保存在私有 S3 兼容对象存储 Bucket；CSV 导入导出不使用对象存储。
- 浏览器只使用服务端签发的短时 URL 直传或下载。签名 URL、Access Key、Session Token 和签名请求头不得持久化到业务表、日志、审计或前端存储。
- CMS 数据库保存稳定对象 Key，不保存对象存储 URL。所有对外下载请求每次重新鉴权并生成新签名。
- 生产启动必须验证 Endpoint 和 Bucket 可访问；提供方支持 Location/ACL API 时同时验证 Region 和公共授权；始终以实际匿名读写请求确认 Bucket 拒绝公共访问，并验证凭证可执行 `PutObject`、`HeadObject`、`GetObject` 和受限删除操作。检查失败时启动失败。

生产配置固定为：

| 环境变量 | 必填 | 规则 |
| --- | --- | --- |
| `S3_ENDPOINT` | 是 | S3 兼容 HTTPS Endpoint，不接受用户请求覆盖 |
| `S3_REGION` | 是 | 与 Bucket 所在 Region 一致 |
| `S3_BUCKET` | 是 | 私有 Bucket 名称 |
| `S3_ACCESS_KEY_ID` | 是 | 仅应用入口读取 |
| `S3_ACCESS_KEY_SECRET` | 是 | 仅应用入口读取，不得输出 |
| `S3_SESSION_TOKEN` | 否 | 使用临时凭证时填写，并与 Access Key 同生命周期 |
| `S3_USE_PATH_STYLE` | 否 | 默认 `false`；仅兼容实现明确要求时设为 `true` |
| `S3_BUCKET_ENDPOINT` | 否 | 默认 `false`；Endpoint 已绑定单个 Bucket 的 CNAME 场景设为 `true`，不得与 path style 同时启用 |
| `S3_UPLOAD_URL_TTL` | 否 | 默认 15 分钟，允许 1 至 30 分钟 |
| `S3_DOWNLOAD_URL_TTL` | 否 | 默认 5 分钟，允许 1 至 15 分钟 |

生产环境不得使用仓库默认凭证、公共 Bucket、HTTP Endpoint 或无限期签名。应用不负责创建 Bucket、修改 Bucket ACL 或签发跨 Bucket 权限。

阿里云部署使用官方 S3 兼容 Endpoint和 virtual-host style，即 `S3_USE_PATH_STYLE=false`；Bucket 绑定 CNAME 时改用 CNAME Endpoint，并设置 `S3_BUCKET_ENDPOINT=true`。浏览器直传的 Bucket CORS 必须允许 CMS Origin、`PUT` 及 `Content-Type`、`x-amz-meta-sha256` Header；对象存储支持标准条件写时还需允许 `If-None-Match`。

### 1.2 S3 Adapter

素材和导入导出模块只依赖提供方无关的 Adapter。Adapter 使用 AWS SDK for Go v2 的 S3 客户端和 PresignClient，不得手写请求签名；SDK 只存在于 Adapter 实现。公开语义冻结如下，具体 Go 包名可按领域所有权落位：

```go
type ObjectStore interface {
	SignPut(ctx context.Context, req SignPutRequest) (SignedRequest, error)
	Head(ctx context.Context, objectKey string) (ObjectMetadata, error)
	SignGet(ctx context.Context, req SignGetRequest) (SignedRequest, error)
	Put(ctx context.Context, req PutObjectRequest, body io.Reader) (ObjectMetadata, error)
	Get(ctx context.Context, objectKey string) (io.ReadCloser, ObjectMetadata, error)
	Delete(ctx context.Context, objectKey string) error
}

type SignPutRequest struct {
	ObjectKey   string
	ContentType string
	Size        int64
	SHA256      string
	ExpiresAt   time.Time
}

type SignGetRequest struct {
	ObjectKey         string
	DownloadFilename string
	ExpiresAt         time.Time
}

type PutObjectRequest struct {
	ObjectKey   string
	ContentType string
	Size        int64
	SHA256      string
}

type SignedRequest struct {
	Method    string
	URL       string
	Headers   map[string]string
	ExpiresAt time.Time
}

type ObjectMetadata struct {
	ObjectKey    string
	Size         int64
	ContentType  string
	SHA256       string
	ETag         string
	LastModified time.Time
}
```

- `SignPut` 只签发 `PUT`，并把 `Content-Type` 和标准 S3 用户元数据 `Metadata["sha256"]` 纳入签名；该元数据在 HTTP 中表示为 `x-amz-meta-sha256`。启动探测确认对象存储支持标准条件写时还必须绑定 `If-None-Match: *`，阻止签名 URL 重放覆盖；兼容服务返回明确的不支持响应时自动降级，继续依赖高熵对象 Key 和确认阶段的完整性校验。浏览器不可控的 `Content-Length` 不得签名或作为前端必设 Header，大小由确认阶段的 `Head` 与受限流式 `Get` 校验；`SignGet` 只签发 `GET`。
- `ExpiresAt` 必须由调用方按类型化 TTL 计算，Adapter 拒绝过去时间或超过配置上限的时间；`DownloadFilename` 只用于安全生成 `Content-Disposition`，不得改变对象 Key。
- `Put` 必须按 `Size` 限制读取并提交规范 `Content-Type` 和 S3 用户元数据 `Metadata["sha256"]`，实际读取字节数不等于 `Size` 时失败并清理未完成对象。
- `Head` 的对象不存在错误必须可与临时对象存储故障区分。网络、限流和 S3 兼容服务 5xx 映射为可重试 `object_store_unavailable`；签名或权限配置错误映射为不可重试内部配置错误，不向调用方暴露凭证细节。
- `Put` 和 `Get` 用于素材对象读写。`Delete` 只清理尚未确认的素材上传对象，不用于物理删除已确认素材对象。
- `ObjectMetadata.SHA256` 来自已签名的 S3 用户元数据 `Metadata["sha256"]`，只用于确认上传声明没有被替换。确认素材和导入源文件时，服务端还必须通过 `Get` 流式计算对象实际 SHA-256 并与申请值比较，不能仅信任客户端可写的对象元数据。V1 不把 ETag 当作 MD5 或 SHA-256；ETag 仅作为独立的不透明元数据保存和比较。
- Adapter 的 Fake 必须覆盖对象不存在、元数据不匹配、签名过期、临时故障和流式读写，不依赖真实企业对象存储。

## 2. 素材对象、Key 和状态

### 2.1 对象 Key

- 对象 Key 只能由服务端生成，固定格式为 `assets/{asset_id}/{32位小写十六进制随机值}`。`asset_id` 和随机值均在创建上传申请时确定。
- 用户文件名不得出现在对象 Key 中，不得接受调用方传入完整或部分 Key，也不得从文件扩展名推断 MIME。
- Key 是服务端管理的稳定存储定位符。管理端通用素材 DTO、下载响应和审计不返回 Key；客户端内容 API 仅在当前授权发布 Revision 的 `referenced_assets` 中返回对应 Key，供受信任的发布流水线映射或推送 CDN。Key 不是 URL，也不包含 Bucket、Endpoint 或凭据。

### 2.2 素材状态和元数据

素材状态固定为：

```text
quarantined
available
archived
```

- 创建上传申请时即创建 `quarantined` 素材。该状态不可用于内容值、不可发布、不可下载。
- 上传确认成功后原子写入确认元数据并转换为 `available`。确认失败保持 `quarantined`，允许在上传申请过期前重新直传和确认。
- `available -> archived` 是唯一归档转换，不可恢复。归档不物理删除对象，也不改写历史 Revision 或当前已发布 Revision。
- 新建条目或新增素材引用时只能引用 `available` 素材；编辑已有草稿时可从 `base_revision_id` 原样继承该 Revision 已引用的 `archived` 素材，但不能新增或从其他 Revision 引入归档素材。发布前必须在发布事务中重新验证候选 Revision 的全部素材仍为 `available`；任一素材隔离、归档或不存在时返回 `409 asset_not_publishable`，发布指针不切换。
- 已经在线的 Revision 所引用素材后来归档时继续可下载，避免静默破坏线上内容；归档素材不能随新 Revision 再次发布。
- 病毒扫描不属于 V1。`quarantined` 仅表示上传尚未确认，为后续扫描扩展预留状态。

素材固定保存并返回以下非敏感元数据：`id`、`filename`、`mime_type`、`size`、`sha256`、`etag`、`status`、`created_by`、`created_at`、`confirmed_at`、`archived_at`。其中：

- `filename` 是去除路径后的原始显示文件名，长度 1 至 255 个 Unicode 码点，禁止 NUL、`/` 和 `\`。
- `mime_type` 是小写规范 MIME，最长 255 字节；允许列表由类型化配置注入，V1 不接受 `application/octet-stream` 作为绕过值。
- 媒体种类 `image`、`audio`、`video` 由内置规范 MIME 集合判定：图片为 `image/jpeg`、`image/png`、`image/gif`、`image/webp`、`image/avif`，音频为 `audio/mpeg`、`audio/mp4`、`audio/ogg`、`audio/wav`、`audio/webm`，视频为 `video/mp4`、`video/webm`。不从文件扩展名推断；管理列表的 `kind` 也不等同于用于展示能力判定的 `preview_kind`。
- `size` 是字节数，必须为 1 至配置上限；同步确认要求完整读取对象并计算 SHA-256，因此配置上限不能超过 100 MiB。
- `sha256` 是 64 位小写十六进制。客户端在申请上传前计算，上传时作为签名元数据提交；确认时必须与 `Head` 返回值一致。
- `etag` 是去除对象存储响应外围引号后的非空不透明字符串。不得用它替代 SHA-256 完整性字段。

### 2.3 上传申请和确认

上传流程固定为：

1. `POST /api/admin/v1/assets/uploads` 提交 `filename`、`mime_type`、`size`、`sha256`。
2. 服务端校验 `assets.upload`、元数据和配额，创建 `quarantined` 素材并返回 `AssetUpload`。
3. 浏览器按响应中的 `method`、`url` 和全部 `headers` 直接 `PUT`；不得增删已签名头。
4. `POST /api/admin/v1/assets/{asset_id}/confirm` 携带空 JSON object。
5. 服务端 `Head` 固定 Key，逐项比较对象大小、规范 MIME、SHA-256 元数据和非空 ETag，再通过 `Get` 流式计算实际 SHA-256；完全一致后转为 `available`。

- 上传 URL 在 `expires_at` 失效。过期后返回 `409 asset_upload_expired`，调用方必须创建新素材上传申请，不能为同一素材续签。
- 确认是幂等的：已经成功确认且对象元数据仍匹配时返回同一个 `Asset`；归档素材不能确认。
- 对象不存在返回 `409 asset_object_missing`；大小、MIME、声明 SHA-256、实际 SHA-256 或 ETag 不一致返回 `409 asset_metadata_mismatch`；临时对象存储故障返回 `503 object_store_unavailable` 并允许重试。
- 元数据不一致不得自动采用对象存储实际值，不得把素材设为可用。过期或长期未确认对象由清理任务处理。
- 素材确认全进程最多 2 个、同一用户最多 1 个并发请求，不排队；超过并发上限返回 `429 asset_confirm_concurrency_limit`。历史遗留的 100 MiB 以上隔离素材在访问对象存储前返回 `413 asset_confirmation_too_large`。

### 2.4 下载

- 管理端 `GET /api/admin/v1/assets/{asset_id}/download` 要求 `assets.view`，`available` 和 `archived` 可下载，`quarantined` 返回 `409 asset_not_available`。
- 客户端 `GET /api/content/v1/assets/{asset_id}` 要求有效 API Key，并且该素材被至少一个“API Key 授权模型的当前发布 Revision”引用。其他情况统一返回 `404 published_asset_not_found`，不得泄露素材或模型存在性。
- 下载成功返回 `302`，`Location` 是短时对象存储签名 URL，同时返回 `Cache-Control: private, no-store`。响应和审计不得记录 `Location` 完整值。
- 每次下载都重新检查当前权限、API Key 状态、发布指针和素材状态。管理权限或模型范围撤销后，旧 CMS 路由请求立即失效；已经签发的对象存储 URL 只存活到其短 TTL 到期。

## 3. 媒体 JSON 和 Revision 引用

媒体字段内容值固定为：

| 字段类型 | JSON 表示 |
| --- | --- |
| `single_media` | 素材 ID string；缺省或 `null` 表示无素材 |
| `multi_media` | 素材 ID array；顺序有意义，不允许重复，最多 50 项 |

- 富文本媒体是 `RichTextDocument` 的块节点：图片固定为 `{"type":"image","attrs":{"asset_id":"...","alt":"..."}}`，音频固定为 `{"type":"audio","attrs":{"asset_id":"..."}}`，视频固定为 `{"type":"video","attrs":{"asset_id":"..."}}`。`image.attrs.alt` 必须存在且为 string，可为空字符串；节点及 attrs 均 `additionalProperties=false`，不接受 URL、文件名或内嵌二进制替代 `attrs.asset_id`。
- 富文本 `image`、`audio`、`video` 节点分别只能引用上述对应 MIME 集合中的素材。新引用必须为 `available`；基于旧 Revision 编辑时可原样继承该 Revision 已引用的 `archived` 素材，但不可新增归档素材引用。类型不匹配返回 `409 asset_media_type_invalid`，不能由文件扩展名或 `preview_kind` 放宽。
- 媒体字段可位于模型允许的任意深度，包括对象和重复组。数组、对象位置通过 RFC 6901 JSON Pointer 定位，例如 `/gallery/0/image`。
- `asset_references` 是 Revision 级不可变引用，覆盖 `single_media`、`multi_media` 字段和富文本媒体节点。每行至少保存 `revision_id`、源 `entry_id/model_id`、`field_id`、`asset_id`、字段值或富文本节点 `attrs.asset_id` 的 `json_pointer` 和从 0 开始的 `position`；富文本指针例如 `/body/content/2/attrs/asset_id`。
- 单媒体固定 `position=0`；多媒体按数组顺序保存。`(revision_id,json_pointer,position)` 唯一，同一多媒体值中的素材 ID 不得重复。
- 富文本媒体节点固定 `position=0`，节点顺序和位置由唯一 JSON Pointer 表达；同一素材可在文档中出现多次，每个节点均保留独立指针。
- 创建 Revision 时，内容模块先收集全部媒体字段值和富文本媒体节点，通过素材服务批量验证 `available` 及节点 MIME 种类，再在同一 MySQL 事务插入 Revision JSON、投影、关联、素材引用、指针和成功审计。任一步失败整体回滚。
- 历史 Revision 的引用不可更新或删除。草稿指针和发布指针变化不复制引用；可见性通过指针连接目标 Revision 判断。
- 素材模块拥有 `asset_references` 表。内容模块只使用冻结的 `ValidateAvailable`、`InsertRevisionReferences` 和 `ValidatePublishableRevision` 语义接口，不直接访问素材表；接口必须接受调用方显式事务执行器。
- 发布锁顺序延续 F2 的 `model -> entry -> revision/pointer`，之后按 `asset_id ASC` 锁定候选素材并校验状态，避免并发归档造成检查后发布竞态。
- 客户端内容 JSON 的媒体字段和富文本媒体节点继续保留素材 ID；每个发布条目及展开条目的 `referenced_assets` 覆盖其活动媒体字段和富文本 `image`、`audio`、`video` 节点实际引用的素材，以素材 ID 为键去重并返回元数据和稳定 `object_key`。调用方可将 Key 交给受信任的 CDN 发布流水线，或把该键及节点 `attrs.asset_id` 代入 `GET /api/content/v1/assets/{asset_id}` 换取短时下载地址；不返回对象存储 URL，素材下载不参与内容响应 ETag 计算。

## 4. 同步传输边界

- CSV 导入导出直接在管理 HTTP 请求内完成，不创建后台任务，不写任务表，也不使用对象存储。
- 导入最多 1,000 行且文件最大 10 MiB；任一行失败时整批事务回滚。
- 导出先写入请求生命周期内的受限临时文件，再设置 `Content-Length` 传输；请求结束后立即删除，不保留可复用结果文件。
- 模板和导出要求目标模型 `content.view`，导入要求目标模型 `content.create`，不新增传输系统权限。

## 5. CSV 编码

### 5.1 文件和表头

- CSV 输入为 UTF-8，可带或不带 UTF-8 BOM；输出固定带 UTF-8 BOM。输入包含非法 UTF-8、NUL 或 BOM 出现在文件开头以外时失败。
- 分隔符固定为逗号 `,`，记录分隔接受 `LF` 和 `CRLF`，输出固定 `CRLF`。字段引用和双引号转义遵循 RFC 4180。
- 第一行是表头，必须与当前模型的全部 active 根字段 key 完全相同，顺序固定为模型字段定义顺序。不得使用显示名称，不得缺列、增列、重复列或重排。
- 模板只包含 BOM 和表头，不包含示例数据。单文件最多 1,000 个数据行，表头不计入；超过后以 `row_limit_exceeded` 失败。
- CSV 文件最大 10 MiB。解析器逐行读取，并在进入事务前将最多 1,000 条规范化内容保存在内存中，以支持批量预检和正式写入两次遍历。

### 5.2 各字段编码

空 CSV 单元格表示字段缺失；非空单元格按下表解析。JSON 字面量 `null` 只对 `json`、`object` 和 `repeatable_group` 表示实际 JSON null。必填字段缺失仍按普通内容校验失败。

| 字段类型 | CSV 单元格编码 |
| --- | --- |
| `single_line_text`、`multi_line_text` | JSON string；空字符串写为 `""`，从而与缺失单元格区分 |
| `rich_text` | 符合 F1/F3 `RichTextDocument` 的紧凑 JSON object string；媒体节点原样编码 `attrs.asset_id`，图片同时编码必填 `attrs.alt` |
| `integer` | 无前导加号的规范十进制整数 |
| `decimal` | F1 规范十进制 string，不接受指数 |
| `boolean` | 小写 `true` 或 `false` |
| `date` | `YYYY-MM-DD` |
| `datetime` | RFC 3339，导入后归一化 UTC |
| `single_select` | 稳定枚举 `value` |
| `multi_select` | 枚举 `value` 的紧凑 JSON string array |
| `json` | 任意合法紧凑 JSON value string |
| `single_media` | 素材 ID string |
| `multi_media` | 素材 ID 的紧凑 JSON string array |
| `single_relation` | 目标内容条目 ID string |
| `multi_relation` | 目标内容条目 ID 的紧凑 JSON string array |
| `object` | 以 child key 为属性名、值继续遵循字段 JSON 编码的紧凑 JSON object string |
| `repeatable_group` | 以 child key 为属性名的紧凑 JSON object array string |

- “紧凑 JSON”指无语义外空白的 UTF-8 JSON；对象属性按模型 child 定义顺序输出，导入接受任意属性顺序但拒绝重复键、未知键和非规范字段值。
- 多值数组顺序有意义并禁止重复；关联和媒体 ID 都是不透明字符串，不接受 URL、文件名、模型 key 或显示名称替代。
- 富文本媒体节点按第 3 节校验严格属性、素材状态和 MIME 种类，并与普通媒体字段一起写入 Revision 的 `asset_references`；任一节点失败整行失败。导出保留节点顺序、`attrs.asset_id` 和图片 `attrs.alt`，不得展开为 URL 或 `referenced_assets` 元数据，保证可无损重新导入。
- 导出缺失值输出空单元格；JSON 实际 `null` 输出 `null`。所有其他值按上表规范编码，再按 RFC 4180 执行 CSV 引号转义，保证导出文件可作为同一未破坏性变更模型的导入输入。

## 6. CSV 导入

### 6.1 上传和创建

- `POST /api/admin/v1/models/{model_id}/imports` 要求目标模型 `content.create`，接收仅含一个 `file` 字段的 `multipart/form-data`。
- 服务端限制文件为 10 MiB、数据为 1,000 行，并按当前模型全部 active 根字段解析表头和单元格。
- 成功返回导入数量；失败返回包含 `row`、`field`、`code`、`message` 的结构化详情。

### 6.2 暂存、全量校验和正式写入

导入固定执行以下阶段：

1. 在请求内逐行解析 CSV，将规范化内容保存在受上限约束的内存切片中。
2. 调用内容模块批量导入语义，对模型、类型、必填、唯一性、关联、媒体状态和对象结构执行预检。
3. 全部行通过后，在一个 MySQL 事务中创建草稿条目、首个不可变 Revision、投影、关联、素材引用、唯一占用和成功审计。
4. 任意错误使整个事务回滚，不保留部分成功内容。

- 导入上限固定为 1,000 行，以限制同步请求时长和数据库事务规模。
- 导入不能调用提交、审核或发布动作，所有新 Revision 状态为 `draft`。
- 正式写入前必须重新校验与现有正式数据相关的唯一值、关联和素材状态，防止暂存校验后的并发变化。

### 6.3 错误响应

- 错误详情固定为 `row`、`field`、`code`、`message`。`row` 是 CSV 中从 2 开始的物理记录号；文件级错误使用 `row=1`，`field` 为空字符串。
- 当前响应返回解析错误或内容服务返回的确定校验详情，不生成独立错误报告文件。
- 稳定导入错误代码至少包括 `invalid_csv`、`invalid_header`、`row_limit_exceeded`、`invalid_encoding`、`validation_failed`、`not_unique`、`relation_not_found` 和 `asset_not_available`。

## 7. CSV 导出

- `GET /api/admin/v1/models/{model_id}/exports.csv` 要求目标模型 `content.view`，接受与管理列表同语义的 `workflow_status`、`filter`、`relation_filter` 和 `sort`。
- 导出按稳定排序分页读取当前草稿内容，先写入请求生命周期内的临时文件，再设置 `Content-Length` 传输，首字节固定 UTF-8 BOM，表头和字段编码遵循第 5 节；请求结束后立即删除临时文件。
- 单次导出最多 10,000 行、50 MiB；全进程最多 2 个、同一用户最多 1 个并发导出；生成和传输总期限 120 秒。导出不创建任务、不写对象存储、不保留可复用结果文件。
- 超过行数或文件大小分别返回 `413 export_row_limit_exceeded`、`413 export_size_limit_exceeded`；超过并发额度返回 `429 export_concurrency_limit`；生成在 120 秒内未完成返回 `503 export_timeout`。传输已开始后的客户端断开或超时只能终止连接并记录错误，不能改写已提交状态。

## 8. 管理和客户端路由

### 8.1 管理素材

```text
GET  /api/admin/v1/assets
POST /api/admin/v1/assets/uploads
GET  /api/admin/v1/assets/{asset_id}
PATCH /api/admin/v1/assets/{asset_id}
DELETE /api/admin/v1/assets/{asset_id}
POST /api/admin/v1/assets/{asset_id}/confirm
GET  /api/admin/v1/assets/{asset_id}/download
```

- 列表、详情和下载要求 `assets.view`；上传申请和确认要求 `assets.upload`；`PATCH` 只允许修改 `filename` 并要求 `assets.update`；归档要求 `assets.archive`。
- 列表可按状态、规范 MIME 精确值和 `kind=image|audio|video` 筛选，按 `created_at DESC,id DESC` 使用 F1 游标外壳。`kind` 按第 2.2 节内置规范 MIME 集合匹配，不按扩展名或 `preview_kind` 匹配；与 `mime_type` 同时提供时取交集。
- 归档成功返回 `204`；重复归档返回 `409 resource_archived`。归档动作、上传确认和下载按设计写入审计，审计不含对象 Key或签名 URL。

### 8.2 管理传输

```text
GET  /api/admin/v1/models/{model_id}/transfers/template
POST /api/admin/v1/models/{model_id}/imports
GET  /api/admin/v1/models/{model_id}/exports.csv
```

- 模板要求目标模型 `content.view`，同步返回 `text/csv; charset=utf-8` 文件。
- 导入要求目标模型 `content.create`；导出要求目标模型 `content.view`。

### 8.3 客户端素材

```text
GET /api/content/v1/assets/{asset_id}
```

客户端 API 不提供素材列表、元数据详情、上传、归档或任意写路由。成功仅返回 `302`，不返回 `Asset` DTO。

客户端素材路由只由生产集成层 `ClientAssetHandler` 注册；素材模块 Handler 只注册 8.1 的管理路由，不持有客户端 API Key scope provider。

## 9. 路由 DTO 和稳定错误码

统一错误外壳沿用 F0，校验详情沿用 F1 `{path,code,message}`。F3 新增稳定顶层错误码：

| HTTP | `error.code` | 适用情况 |
| --- | --- | --- |
| 400 | `validation_failed` | 素材元数据、导入导出请求或 CSV 字段校验失败 |
| 400 | `invalid_cursor` | 素材列表游标无效 |
| 400 | `invalid_query` | 导出筛选参数无效 |
| 401 | F1/F2 既有认证码 | 管理会话或 API Key 无效 |
| 403 | `permission_denied` | 管理用户缺少素材或模型权限 |
| 404 | `resource_not_found` | 管理端素材、上传或模型不存在 |
| 404 | `published_asset_not_found` | 客户端素材不存在或没有授权发布引用 |
| 409 | `resource_archived` | 重复归档或修改归档素材 |
| 409 | `asset_upload_expired` | 上传申请已过期 |
| 409 | `asset_object_missing` | 确认时对象存储对象不存在 |
| 409 | `asset_metadata_mismatch` | 确认时大小、MIME、SHA-256 或 ETag 不符合申请 |
| 409 | `asset_not_available` | 管理下载、引用或确认遇到不可用状态 |
| 409 | `asset_not_publishable` | 候选 Revision 含非 available 素材 |
| 413 | `file_too_large` | 申请文件或实际对象超过配置上限 |
| 413 | `asset_confirmation_too_large` | 历史隔离素材超过同步确认上限 |
| 413 | `export_row_limit_exceeded` | 同步导出超过 10,000 行 |
| 413 | `export_size_limit_exceeded` | 同步导出超过 50 MiB |
| 429 | `asset_confirm_concurrency_limit` | 素材确认达到进程或用户并发上限 |
| 429 | `export_concurrency_limit` | CSV 导出达到进程或用户并发上限 |
| 429 | `content_rate_limited` | 内容 API Key 或 Key/IP 请求过于频繁 |
| 429 | `content_concurrency_limited` | 内容 API Key 或 Key/IP 并发请求过多 |
| 503 | `content_api_busy` | 内容 API 全进程并发或限流状态容量达到上限 |
| 503 | `export_timeout` | CSV 导出生成超过 120 秒期限 |
| 503 | `object_store_unavailable` | 对象存储临时不可用，可安全重试 |

- `object_store_unavailable` 可带 `Retry-After`，调用方可以重试素材请求。
- CSV 错误详情直接放入 `ErrorResponse.details`，数量受同步导入规模限制。
- `302` 的 `Location`、`304` 和成功 CSV 文件响应不是错误，不返回 `ErrorResponse`。

## 11. 实现与兼容门禁

- 验证对象 Key 不含用户文件名，上传签名绑定方法、MIME、`If-None-Match` 和 SHA-256 元数据，不绑定浏览器不可控的 `Content-Length`；确认逐项比较对象元数据并流式计算实际 SHA-256。
- 验证私有 Bucket 启动探测、签名过期、对象不存在和可重试故障映射，且凭证、Key 和签名 URL 不进入日志、审计或前端产物。
- 验证单媒体、多媒体、嵌套媒体、Revision 引用不可变性，以及 JSON、投影、关联、素材引用、指针和审计事务一致性。
- 验证候选 Revision 发布前重新校验素材；归档不改写历史和已发布内容，客户端只可下载授权发布 Revision 引用的素材。
- 使用 1,000 行边界 CSV 验证大小和行数限制、全量校验、任一行失败正式内容零写入，以及成功时完整写入。
- 验证所有字段 CSV 编码、严格表头、UTF-8/BOM、RFC 4180 引号、对象与重复组 JSON、行级错误排序和截断。
- 验证导出筛选、稳定排序、UTF-8 BOM 和 HTTP 流式下载。
- F3 分片只有在实现、契约测试和集成 Agent 评审完成后，才可移除 `pending-implementation` 并聚合进根 OpenAPI。
- F3 完成后，本文路由、方法、DTO 必填字段、素材状态、媒体 JSON、CSV 编码和稳定错误码只能做兼容扩展；破坏性修改必须形成新冻结决议。
